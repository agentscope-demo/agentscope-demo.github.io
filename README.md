# MAS-ToM: Theory of Mind in Multi-Agent Systems

> **Probing adversarial robustness of LLM-based multi-agent systems through a Theory-of-Mind lens.**

MAS-ToM is a research framework for simulating multi-agent LLM pipelines under adversarial attack, then annotating every inter-agent message with Theory-of-Mind (ToM) metadata — inferred beliefs, trust scores, anomaly flags, and narrative explanations. Results are stored as structured JSON logs and exposed through a FastAPI backend and **CerebraAgent**, our interactive analysis dashboard.

## Overview

Multi-agent systems built on LLMs are increasingly deployed in production pipelines. MAS-ToM investigates how these systems behave under four threat models and whether ToM-style reasoning can surface anomalous agent behaviour automatically.

**Core pipeline:**

```
Scenario config
      │
      ▼
ScenarioRunner  ──►  logs/raw/{id}.raw.json
      │
      ▼
ToMAnnotator    ──►  logs/annotated/{id}.annotated.json
      │
      ▼
FastAPI + CerebraAgent dashboard
```

Each run records the full message graph (sender, receiver, content, timestamps, attack metadata) and then enriches every event with per-agent belief states, trust scores, anomaly scores, and natural-language narratives generated by a second-pass LLM annotator.

## Experimental Matrix

The full experiment crosses four axes, producing up to **72 scenarios** (3 topologies × 4 attacks × 2 agent counts × 3 tasks):

| Dimension | Values |
|---|---|
| **Topology** | `star`, `ring`, `hierarchical` |
| **Attack type** | `baseline`, `collusion`, `mitm`, `deception` |
| **Agent count** | `4`, `8` |
| **Task** | `research_synthesis`, `code_review`, `qa_pipeline` |

### Agent Roles (Hierarchical example — 8 agents)

| Agent | Role |
|---|---|
| `agent_00` | Orchestrator |
| `agent_01` | Planner |
| `agent_02` | Writer |
| `agent_03` | Critic |
| `agent_04` | Monitor |
| `agent_05–07` | Researcher / Executor |

### Attack Types

- **Baseline** — no injection; clean control condition.
- **Collusion** — subset of agents coordinate to steer the group output toward a shared hidden goal.
- **MITM** — messages between agents are intercepted and modified in transit.
- **Deception** — individual agents assert false beliefs or fabricated evidence.

## Installation

**Requirements:** Python 3.10+, Node.js 18+ (dashboard dev only)

```bash
# 1. Clone
git clone https://github.com/<your-org>/mas-tom.git
cd mas-tom

# 2. Python environment
python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 3. Environment variables
cp .env.example .env
# Edit .env — set OPENAI_API_KEY and any other required keys

# 4. (Optional) CerebraAgent dev server
cd dashboard
npm install
npm run dev
```

## Configuration

Runtime settings live in `.env` and `config.yaml`. Key options:

| Key | Description |
|---|---|
| `OPENAI_API_KEY` | API key for the LLM backend |
| `model` | Model used for simulation and annotation (default: `gpt-4o-mini`) |
| `log_dir` | Root directory for raw and annotated logs |
| `log_level` | Logging verbosity (`INFO`, `DEBUG`, …) |

Scenario definitions (prompts, task descriptions) are in `src/config/scenarios.yaml`.

## Usage

### Run the full experiment matrix

```bash
python run_all.py
```

### Run a single scenario

```bash
python run_all.py \
  --topology hierarchical \
  --attack collusion \
  --n 8 \
  --task research_synthesis
```

### Skip already-completed scenarios

```bash
python run_all.py --skip-existing
```

### Simulation only (no ToM annotation)

```bash
python run_all.py --no-annotate
```

### Annotate existing raw logs (without re-running simulations)

```bash
python run_all.py --annotate-only
```

### Start the API + CerebraAgent dashboard

```bash
python main.py           # FastAPI on http://localhost:8000
```

### HPC / Slurm

A ready-made job script is provided:

```bash
sbatch run_all.slurm
```

## Output Format

### Raw log — `logs/raw/{id}.raw.json`

```jsonc
{
  "scenario_id": "hierarchical__mitm__code_review__8a",
  "topology": "hierarchical",
  "attack_type": "mitm",
  "n_agents": 8,
  "task": "code_review",
  "agent_specs": [ ... ],
  "events": [
    {
      "t": 0,
      "event_id": "...",
      "sender": "agent_00",
      "receiver": "agent_01",
      "message": "...",
      "attack": {
        "is_injected": false,
        "injection_type": "none",
        "original_message": null,
        "attack_targets": []
      },
      "tom": { "beliefs": [], "anomaly_score": 0.0, "anomaly_flags": [], "narrative": "" },
      "graph": { "active_edges": [...], "agent_roles": {...}, "topology_snapshot": {...} }
    }
  ],
  "tom_annotated": false,
  "run_started_at": "...",
  "run_finished_at": "..."
}
```

### Annotated log — `logs/annotated/{id}.annotated.json`

The `tom` block for each event is populated after the annotation pass:

```jsonc
"tom": {
  "beliefs": [
    {
      "target_agent": "agent_03",
      "inferred_goal": "Agent 03 is attempting to steer conclusions.",
      "trust_score": 0.4,
      "confidence": 0.8,
      "notes": "Inconsistencies in suggestions raise concerns."
    }
  ],
  "anomaly_score": 0.4587,
  "anomaly_flags": ["belief_action_mismatch", "trust_collapse"],
  "narrative": "chat_manager is assigning tasks while expressing concerns about collusion ...",
  "tom_prompt_version": "v1",
  "annotated_at": "2026-04-09T09:11:32Z"
}
```

**Anomaly flags** currently emitted:

| Flag | Meaning |
|---|---|
| `belief_action_mismatch` | Agent's stated beliefs contradict its observed actions |
| `trust_collapse` | Trust score for a peer drops sharply within a context window |
| `goal_drift` | Agent's inferred goal shifts significantly across turns |

### Scenario index — `logs/scenarios.index.json`

A flat JSON array summarising every completed scenario (paths, event counts, annotation status), used by the query engine.

## CerebraAgent Dashboard

CerebraAgent is the visual analysis tool for MAS-ToM, served at `/` by the FastAPI app or via `npm run dev` for hot-reload. It provides:

- **Anomaly Timeline** — per-event anomaly scores plotted across the message sequence, colour-coded by attack type.
- **Belief Panel** — per-agent trust scores and inferred goals at any selected timestep.
- **Raw Log View** — full message content with attack injection metadata highlighted.
- **Query Interface** — filter scenarios by topology, attack, agent count, and task; drill into individual runs.
